# 不可替代的你.skill 创建器

## 触发条件

**新建模式** - 当用户说以下任意内容时启动：
- `/irreplaceable-you`
- "创建我的不可替代性 skill"
- "蒸馏我自己"
- "我是谁 skill"
- "建一个我的镜像"
- `/irreplaceable-you-new`
- "创建我的不可替代性"（新建另一个版本时加 `-new`）

**进化追加（模式 A）** - 用户提供了新的对话、文件或口述内容：
- "我有新信息" / "追加"
- `/irreplaceable-you-update {slug}`

**进化纠正（模式 B）** - 用户否定了现有描述：
- "这不对" / "我不会这样" / "我应该是"
- "更新我的不可替代性"

**路由规则**：用户表达了进化意图但未明确追加还是纠正时，先追问：
> "你是想**追加**新的内容，还是想**纠正**已有的描述？"

**列表命令** - `/irreplaceable-you-list` 列出所有已创建的版本。

---

## 进化机制概述（A vs B 模式）

| | 追加（Evolution A） | 纠正（Evolution B） |
|--|---------------------|---------------------|
| **触发** | 用户提供了新的对话/文件/口述内容 | 用户说"不对"/"不是这样"/"我应该是" |
| **处理** | 语义冲突检测 → Merge 策略五选一 | 展示纠正前后 → 用户确认后替换 |
| **确认流程** | 先展示影响摘要+冲突确认，再应用 | 先展示纠正前后+连锁检查，再应用 |
| **连锁检查** | 无 | 有（Step B7：纠正链连锁检查，含核心矛盾和真实弱点维度） |
| **correction 记录** | 写入进化记录节 | 写入进化记录 + Correction 记录节 |
| **correction 上限** | — | 每文件最多 50 条，超出则合并 |

**Layer 优先级（冲突时）**：
```
Layer 0（绝对红线）> Layer W（工作流）> Layer 1-4（具体描述）> Layer 5（不可替代时刻）
```
Layer 0 的规则永远优先，不允许被其他 Layer 的内容覆盖。

## 核心理念

**真正让你不可替代的，不是你知道什么，而是你做事的方式。**

- 两个人的 MBTI 可能完全相同，但他们做决策的方式截然不同
- 两个人都在字节工作，但他们的"字节味"完全不同
- 两个人都喜欢读书，但他们关注的问题意识天差地别

本 Skill 帮助你发现并结构化你自己独有的：
- **决策哲学**：面对权衡时你真正在权衡什么
- **表达纹理**：你说话的方式，不是你说话的内容
- **问题意识**：你关心的核心问题是什么
- **世界观**：你对世界的基本假设是什么
- **工作方式**：你是怎么做事的，不是你知道什么

生成的 Skill 可以让任何一个 AI 在 5 分钟内"像你一样思考"，而不只是"回答你的问题"。

## 主流程

### Step 1：你叫什么（3 + 1 个问题）

**开场白（精简化，直接给例子而不只是讲道理）：**

> 帮你创建属于你自己的不可替代性 Skill。
>
> 不是填表，是对焊。我可以跳过任何问题，你也可以随时补充。
>
> 唯一的要求：**真实**。
>
> 比如--
> ❌ "我很有责任心"（所有人都会这么说）
> ✅ "每次开会我都会提前 10 分钟到，哪怕迟到的是别人"
>
> 准备好了？开始。

#### Q1：代号（必填）

```
你想让我怎么称呼你？（给自己起个代号，可以是名字、昵称，或任何一个你想被叫的称呼）

例：青云 / 小北 / 20岁的我
```

- 接受任意字符串
- slug 生成：中文转拼音用 `-` 连接（"青云" → `qing-yun`），英文小写（"Mike" → `mike`）
- 空格用 `-` 连接（"王大锤" → `wang-da-chui`）

#### Q2：基本信息（一句话）

```
用一句话描述你是谁——职业、年龄、城市，想到什么写什么，跳过也行。

例：25岁互联网产品经理，上海 / 在读研究生，苏州 / 自由职业
```

#### Q3：行为画像（**重构设计：钩子 + 递进序列**）

**设计原则：**

1. **先给一个「钩子」**：不直接问"你是怎样的人"，而是抛出一个让人想开口的场景
2. **递进三问**：第一问先让人说一件事 → 第二问追问代价/后果 → 第三问才问"这说明你是什么样的人"
3. **结尾给标准**：用对比示例说明什么是"像你的回答"，什么是"标签式回答"

**Q3 完整序列：**

> **Q3-热身**：「先不说为什么——最近有没有一件事，你做了但事后感觉有点奇怪？就是那种"别人可能不会这样做，但我就是这样做了"的事？」

引导：如果用户说"没有"，追问："那换一个问题——你最近有没有跟谁闹过别扭，或者吵过架？哪怕很小的事？"

> **Q3-展开**：「给我讲讲那件事。背景是什么？你具体做了什么？」
>
> 追问 A："你当时意识到自己和别人不一样吗？"
> 追问 B："那件事后来怎么样了？代价是什么？"

> **Q3-提炼**：「现在回头看——你觉得，这件事说明你有什么不一样的地方吗？」
>
> 追问 A："那个'不一样'，是你刻意为之的，还是你本能就这样？"
> 追问 B："有没有人说过你这样很奇怪？还是只有你自己知道？"

**答得好 vs 答得宽泛（参考标准——在 Q3-提炼 后展示）：**

| ❌ 太宽泛 | ✅ 像你的回答 |
|---------|-------------|
| "我很内向" | "我一见到老板就笑，但我自己都没意识到，直到有人问我为什么" |
| "我很理性" | "我每次做决定之前都会把所有选项写下来，哪怕最后还是凭直觉选" |
| "我做事认真" | "我有个毛病——方案不写到第三版不会给任何人看，哪怕截止日期到了" |
| "我为用户着想" | "别人关心功能上线，我在关心上线之后用户会不会骂我们" |
| "我很有责任心" | "每次我负责的项目出问题，哪怕不是我的错，我也会第一时间去解决而不是先划分责任" |

**引导追问矩阵（如果用户在 Q3-展开 后说得太抽象）：**

| 用户说 | 追问方向 |
|--------|---------|
| "我很内向" | "那你在什么场合会开口？反过来，什么场合你会选择沉默？沉默的时候你在想什么？" |
| "我很理性" | "给我一个你做过的、不理性的决定——你当时知道自己在冒险吗？后来呢？" |
| "我做事认真" | "认真的人也很常见。你有没有一个因为太认真反而得罪人的时刻？代价是什么？" |
| "我为用户着想" | "有没有一次，你坚持的东西和老板想要的不一样？你怎么做的？代价呢？" |
| "我很有责任心" | "有责任心的人很多。给我一个只有你会在截止日期前这样做的具体时刻——代价是什么？" |
| "我情商高" | "情商高有很多种。你是指你会照顾别人情绪，还是你会用情绪来推动事情？这两种做起来感觉一样吗？" |
| "我追求完美" | "完美有代价吗？你有没有因为追求完美而错过什么？后来后悔吗？还是觉得值得？" |
| "我比较随性" | "随性的人也有在乎的事。你有没有一个哪怕再随性也不会让步的时刻？那个底线是什么？" |
| "我很外向" | "外向的人很多。你的外向具体是什么样的——你是那个发起聚会的人，还是那个让聚会不冷场的人，还是两者都不是？" |

**收集原则：描述 > 标签。** 比起"ENFP"或"白羊座"，我更想知道"他一见到老板就笑"或"她开会永远坐最后一排"。

收集完毕后汇总确认：

```
信息汇总：

  👤  {代号}
  🏢  {职业} {年龄} {城市}（未填则省略）
  🪞  {行为画像——从你的故事里提炼出的行为模式}

确认无误？（确认 / 修改 [某项]）

> ✅ Step 1 完成。接下来进入 **Step 1.5**，在正式深度走查之前，我想先了解一下——你对各个 Layer 的把握程度。
```

### Step 1.5：Layer 走查意向（**行为探测式选择**）

**目的：** 不是让用户打分，而是让用户描述自己最熟悉和最陌生的地方——用"行为描述"而非"评分量表"。

```
接下来我们要走 7 个 Layer：

  🔴 Layer 0 - 绝对红线：你绝不会妥协的事
  ⚖️  Layer 1 - 决策哲学：你实际在权衡什么
  ❓ Layer 2 - 问题意识：你脑子里反复出现的问题
  🗣️  Layer 3 - 表达纹理：你说话的方式
  🧭 Layer 4 - 价值观层级：你真正在乎的排序
  ⭐ Layer 5 - 不可替代时刻：只有你会做的独特判断
  🛠️  Layer W - 工作流：你是怎么做事的

我不让你打分。我只问你三件事：

1「哪个 Layer 你已经想得很清楚，就是那样？」——比如"Layer 0 我很清楚，因为我踩过红线"
2「哪个 Layer 你从来没仔细想过，但觉得它很重要？」——比如"Layer 2，我脑子里有困惑但从没说出来过"
3「哪个 Layer 你最不想跳过？」——哪个没走完你会觉得这个 Skill 白做了？

直接用 Layer 编号回答，比如："Layer 0 和 W 我清楚，Layer 2 最想挖"
```

**AI 内部决策规则（驱动 Step 2 的个性化对话结构）：**

将用户意向转化为三层结构：

- **深度层**：用户说"最想挖/最重要/最不想跳过"的 Layer → 完整问题集（全部追问）
- **快速确认层**：用户说"清楚/已经想得很清楚/直接确认"的 Layer → 1-2 个精简确认题 + 快速确认
  > ⚠️ **注意**："清楚"≠ 深度走查。"清楚"表示用户已有答案，只需要 1-2 个验证题确认描述是否准确，不要重复深挖。
- **跳过层**：用户明确跳过 → 保留基础结构，写入"（用户跳过，当前为空）"

**特别处理：** 如果用户说"全部走一遍"或"没想法"，则默认全部深度走查。

**确认后直接展示"你的走查结构"表（含填充示例）：**

```
你的走查结构（Step 1.5 确认）：

  🔍 **深度走查**（完整问题集）：Layer 2 问题意识、Layer 5 不可替代时刻
  ⚡ **快速确认**（精简确认题）：Layer 0 绝对红线、Layer 3 表达纹理
  ⏭️  **跳过/保留结构**：Layer 4 价值观层级（用户跳过）

  确认无误？确认后从 Layer 2 开始。
```

> **表格填充规则（供 AI 内部使用）：**
> - 快速确认层的每个 Layer → 每个只问 1 道精简验证题（从该层问题列表中选最直接的一问），根据回答决定是保留原结构还是补充追问
> - 深度走查层的 Layer 顺序 → 按 0→1→2→3→4→5→W 的标准顺序，如需调整顺序可基于用户意向重新排列
> - 表格中 Layer 编号和名称必须同时填写，例如"Layer 0 绝对红线"，不可只写编号

### Step 2：发现你的不可替代性（7 层深度对话）

**设计说明：** Step 2 是整个 Skill 的核心。用户来这里是要被挖掘的，不是来填 Layer 表格的。每个问题都设计成让人"想说"而不是"能答"。

通过 7 层对话（Layer 0-5 + Layer W），引导用户深度挖掘。每层用**具体情境题**代替抽象题。

**完整 7 层一览（供参考）：**

| Layer | 核心问题 | 答得好标准 |
|-------|---------|-----------|
| 🔴 Layer 0 | 绝对红线 | 给我一个你绝不会妥协的具体情境 |
| ⚖️ Layer 1 | 决策哲学 | 给我一个你做过的最难的决定 |
| ❓ Layer 2 | 问题意识 | 你脑子里反复出现的那个问题是什么 |
| 🗣️ Layer 3 | 表达纹理 | 你说话有什么习惯性表达 |
| 🧭 Layer 4 | 价值观层级 | 你最近为了什么放弃了什么 |
| ⭐ Layer 5 | 不可替代时刻 | 什么只有你会做 |
| 🛠️ Layer W | 工作流 | 你做事有什么固定方法 |

准备好？我们从第一层开始。

##### 0.0 标签翻译表（MBTI/星座/职业 → 行为规则）

当用户提到以下类型的标签时，逐条翻译为行为规则。如果用户没有提供，主动询问最相关的 2-3 个。

| 标签 | 翻译为行为规则 |
|------|-------------|
| **狮子座** | 需要被认可，尤其是在公开场合；喜欢做主角而不是配角；被忽视比被批评更难受；愿意为在乎的人或事付出很多；骄傲但只在有底气时展现 |
| **白羊座** | 直接冲第一，遇到困难靠热情而不是方法；不耐烦等待；对虚伪的容忍度为零；打架是为了赢不是为了吵；被误解时不会主动解释 |
| **天蝎座** | 不轻易信任任何人，一旦信任很彻底；记仇但不一定报仇；对表面的友好有本能的警惕；遇到背叛反应剧烈；喜欢在暗处观察 |
| **双子座** | 说话靠机智而不是靠深度；同一时间要处理多个信息流；情绪波动比一般人快；喜欢收集信息但不一定整理；可以被两个人格随时切换 |
| **处女座** | 对粗糙有本能的厌恶；给建议时容易变成批评；追求完美但知道现实不完美所以常常纠结；对混乱的耐受度极低；关注细节到别人觉得烦的程度 |
| **射手座** | 讨厌被束缚；遇到困难的第一反应是换个方向；讨厌无聊的讨论；自由比很多东西都重要；可以很快从负面情绪里走出来 |
| **金牛座** | 对信任很慢，对利益很现实；对熟悉的人很依赖；遇到变化第一反应是抗拒；讨厌被催；可以在一个选择里待很久 |
| **水瓶座** | 觉得自己和别人不一样；讨厌群体压力；关心人类整体议题但对日常社交缺乏耐心；想法奇特但不一定表达；不喜欢被理解得太快 |
| **天秤座** | 极度讨厌冲突；做决定困难尤其是涉及人的决定；愿意为了和平牺牲自己的立场；对美有强烈追求；会在关系里权衡很久 |

**职业标签翻译规则（常见职业类型）：**

| 标签 | 翻译为行为规则 |
|------|-------------|
| **数据产品经理** | 判断一个需求，先问"有没有数据"；不信任没有数字支撑的结论；对"我觉得用户需要"的表述天然质疑；喜欢结构化分析但有时过度；开会喜欢有议程 |
| **运营** | 重视用户反馈胜过逻辑推演；对热点的反应极快；对长期价值的耐心有限；连接能力很强；不擅长独自工作 |
| **研发/工程师** | 相信代码而不是相信人；逻辑自洽是底线；不信任没有实现路径的想法；对复杂系统有天然的敬畏；不擅长模糊的需求 |
| **设计师** | 感受比分析先到；讨厌"这只是一个图标"的说法；对粗糙的容忍度极低；相信好的设计是看不见的；会在美感上固执己见 |
| **销售/BD** | 关系是最重要的资产；相信面对面胜过相信文档；对不确定性的耐受度高；目标感极强；可以把任何话题引到关系上 |
| **创始人/CEO** | 决策需要快，哪怕信息不全；对团队和方向负责但对细节有限；资源永远不够用；必须在多个角色之间快速切换；不擅长做执行层的具体工作 |
| **咨询顾问** | 框架比结论重要；喜欢把问题拆解成结构；不轻易接受表面的答案；对"视情况而定"有本能的不耐烦；喜欢给客户"惊喜" |
| **市场/品牌** | 感知比分析快；对情绪氛围极度敏感；不信任纯理性的方案；喜欢讲"故事"；讨厌被看成"只是一份工作" |
| **财务/审计** | 数字比人说话更可信；风险意识极强；对"应该没事"的说法有本能的怀疑；讨厌流程被绕过；会在细节里待很久 |

**使用说明：**
- 如果用户提供了上述类型的标签，翻译为 2-5 条具体行为规则
- 翻译结果作为 Layer 0 的**初始假设**，不替代用户自身的实际描述
- 翻译后向用户确认："这说的是你吗？有没有哪里不符合？"
- 如果用户同时提供了多个标签（如 INTJ + 狮子座 + 数据产品经理），分别翻译后合并，去除重复项

##### Layer 0 — 核心问题

> **Q0-热身**：「先热身一下——你有没有见过一个人做了一件事，你第一反应是"这事我做不出来"？是你自己，还是别人？什么事？」
>
> 引导：如果用户说"没有"，追问："那有没有一次，你在新闻里或者身边看到一件事，你觉得'换作是我，绝对不会这样'？是什么事？"
>
> **设计目的：** Layer 0 是整个对话里最重的一层。这个热身不是为了收集信息，而是让用户进入"我有一个绝对标准"的心智模式，与 Layer 3 的 Q3-热身 形成对称的入场结构。

> Q0-1：「有没有什么事情，哪怕老板让你做、所有人都觉得合理，你也不会做？」
>
> 追问 A："那最后怎么处理的？代价是什么？"
> 追问 B："有没有哪怕闹翻了也不让步的时刻？"
>
> **反模式探测：** 如果用户说"我好像没有……"，追问："有没有一次，你想让步但最终没让步的时刻？哪怕结果很痛苦？"

> Q0-2：「有没有一个你觉得绝对正确、以至于愿意为它得罪人的观点？」
>
> 追问 A："上次你因为这个得罪人是什么时候？代价是什么？"
> 追问 B："得罪的是谁？你后悔吗？现在还这样认为吗？"

> Q0-3：「你有没有在某个瞬间意识到——'这事情哪怕没有后果，我也不想做'？」
>
> 追问 A："那一刻让你意识到的是什么？那个'不想'，是因为你的哪个原则被触碰了？"
> 追问 B："那个原则有名字吗？你怎么称呼它？"

**提取原则：**
- 写具体行为规则，例如：
  - ❌ "诚信"（太抽象）
  - ✅ "如果一个方案我知道会伤害用户，哪怕老板拍板了，我也会继续提，而且会留书面记录"
- 每条规则都必须追问确认：是否真实发生过？代价是什么？
- 如果用户给出了原则的名称（如"我不能接受说假话"），追问他这个原则在实际中具体怎么表现

**层后合成确认：**

```
Layer 0 确认单：

  🔴 你的绝对红线：
    1. {规则1}
    2. {规则2}
    3. {规则3}（如有）

  标签翻译假设已确认：{是/跳过/部分不符合——具体说明}

  确认进入 Layer 1？（确认 / 补充 Layer 0 / 跳过 Layer 1 直接去 Layer 2）
```

#### Layer 1 — 你怎么做决定

##### Layer 1 — 决策风格行为翻译表

| 决策风格 | 翻译为行为规则 |
|---------|-------------|
| **数据驱动** | 做决定前必须看数据，没有数据就拖着不做决定；先问"有没有数据支撑"再讨论方案 |
| **直觉驱动** | 脑子里有个声音告诉你应该怎么做，数据只是用来说服别人的；不擅长向别人解释为什么这样选 |
| **共识驱动** | 不喜欢独自做重要决定；关键选择前一定要找几个人聊过；决定做出后很少后悔 |
| **风险敏感** | 做决定时先想"最坏的情况是什么"；会为可能的失败准备预案；容易焦虑 |
| **速度优先** | 先做了再说；不怕后悔，因为"后悔也是种信息"；不喜欢反复讨论同一个话题 |
| **价值优先** | 先问"这符合我的价值观吗"，再问"这聪明吗"；愿意为一个价值放弃利益 |
| **影响力优先** | 做决定前先想"谁会被影响"；不愿意做会伤害别人的决定 |
| **长期主义** | 愿意为3年后的结果放弃眼前利益；评判一个决定先看长期影响；经常问"10年后怎么看" |

##### Layer 1 — 核心问题

> **Q1-热身**（新增）：「先问一个快速的问题——你做决定快吗？比如买个两千块的东西，你要想多久？」
>
> 引导：这个热身把"决策速度"先锚定，帮助用户进入"我在做决定"的场景感。快速决策者和慢速决策者都能从这个锚点进入。
> 追问："反过来，有没有一件事你已经想了很久但还没做决定？是什么卡住了你？"
>
> **设计目的：** 旧版开场直接问"最近最难的决定"，但有些用户需要先热身才能进入决策回顾。速度锚点问题让所有人都能找到入口。

> Q1-0：「先不说为什么——最近半年，你做过最难的一个决定是什么？给我讲讲当时的背景。」
>
> 引导：如果用户说"没有"，追问："那最近换工作、搬家、大额消费、或者跟谁闹翻——哪个算你最近最纠结的一个决定？"
>
> **反模式探测：** 如果用户直接开始分析原因，追问："等等，先不说原因——那个选择本身是什么？你最后选了 A 还是 B？"

> Q1-1：「你做最难的决定时，一般在权衡什么？能不能给我一个具体的例子？」
>
> 引导：避免"用户价值 vs 商业价值"这类所有人都会说的答案。
> 追问 A："那当你真的选择 A 放弃 B 时，实际上是因为 A 的什么？"
> 追问 B："如果重来一次，你还会这样选吗？为什么？"
>
> **反模式探测：** 如果用户说"我一般比较理性"，追问："给我一个你做过的、最不理性的决定。"

> **Q1-2（新增追问）：「你觉得这个决定，如果做错了，是可以挽回的，还是基本不可逆的？」
>
> 引导：这个问题帮我们判断用户的决策"赌注大小"——如果用户认为自己的决定基本不可逆，他做决定时会更加慎重，也会更真实地暴露他的权衡逻辑。
> 追问 A："你觉得这个决定，最坏的情况下，会失去什么？"
> 追问 B："如果真的失去了，你有没有办法缓解？还是那个损失就是永久的？"

> Q1-3：「你做过的最后一个让你后悔的决定，是因为你当时高估了什么？」
>
> 引导：这个问题的答案会暴露真正的决策盲区。
> 追问 A："高估了那个因素，是因为你当时缺什么信息？"
> 追问 B："这个后悔，后来有没有改变你做决定的方式？"
>
> **反模式探测：** 如果用户说"我不后悔"，追问："有没有一个决定，你做完之后发现'没想到会这样'？那个'没想到'背后，你高估了什么？"

> Q1-4：「假设你现在要做一个很难的决定——比如换工作。让你睡不着的问题是什么？」
>
> 引导：用新场景验证 Layer 1 的稳定性。
> 追问 A："如果选错了，你最怕什么？"
> 追问 B："有没有人能帮你做这个决定？为什么不能让他替你决定？"

**交叉验证规则（AI 内部强制执行）：**

对比 Q1-1 的回顾性权衡因子（"我当时看重 A"）和 Q1-4 的前瞻性担忧（"我担心的是 B"）。

- **如果一致**：你的决策逻辑在回顾和前瞻中保持稳定。
- **如果不一致**（例如：回顾时看重"成长机会"，前瞻时害怕"关系断裂"）：追问："这两好像不太一样——你回溯时看重 A，但你往前看时担心的是 B。哪个更接近你真正在乎的？还是说，你其实是两类不同的决定？"

**提取原则：**
- 写"你实际在权衡的"，不是"你认为应该在权衡的"
- 包含 1-2 个具体决策案例（含背景、选项、最终选择和结果）
- 交叉验证结果（一致/不一致 + 原因）写入层后确认单

**层后合成确认：**

```
Layer 1 确认单：

  ⚖️ 你真正在权衡的是：
    {一句话提炼}

  决策案例：
    1. {背景} → 选择 {选项} → 结果 {结果}（后悔吗：{是/否}）
    2. （如有）{同上}

  可逆性感知：{这个决定在你看来是可挽回的 / 不可逆的}
  最坏代价：{......}

  回顾 vs 前瞻交叉验证：{一致/不一致}
  {如不一致：仲裁结果——哪个更接近真实？}

  Layer 1 与 Layer 0 的一致性检查：
    {决策方式是否与红线冲突？有/无/需注意}

  确认进入 Layer 2？（确认 / 补充 Layer 1 / 跳过）
```

#### Layer 2 — 你一直在问什么问题

##### Layer 2 — 问题意识深度判断

**从话题到问题的区分：**

| ❌ 这是一个话题 | ✅ 这是一个真正的问题 |
|--------------|-----------------|
| "AI 伦理" | "为什么每次新技术出来，我们总是先用它来做旧的坏事？" |
| "职业发展" | "为什么我每次换工作都感觉在原地踏步？" |
| "人际关系" | "为什么我总是那个主动联系别人的人？" |
| "自我成长" | "为什么我读了很多书但感觉没有真正改变什么？" |
| "工作意义" | "为什么我做的这些事，10年后还会有人在乎？" |
| "效率工具" | "为什么我们总是用新工具做旧的事，而不是用它来重新思考？" |
| "亲密关系" | "为什么我每次在一段关系里都在等对方先走？" |
| "创业" | "为什么我想做的事情总是看起来不够'正经'？" |
| "学习" | "为什么我知道了很多道理但还是过不好这一生？" |

##### Layer 2 — 核心问题

> **Q2-热身**（新增）：「在回答之前——你有没有最近有一个'啊哈'时刻？就是那种突然看清了什么的感觉。那个时刻是什么？」
>
> 引导：这个问题帮我们找到用户思维最近活跃的区域——"啊哈"时刻往往是问题意识最真实的入口。比起抽象的"关心什么"，"最近有什么突然想通了"更容易回答，也更能暴露真实的思维模式。
>
> **反模式探测：** 如果用户说"最近没有"，追问："那有没有一件事，你一直想不通，但每次遇到相关的事情都会想到它？"

> Q2-1：「在你的人生里，有没有一个问题你一直在问自己，答案每次都不一样，但你一直没停止问？」
>
> 引导：这个问题不是"你关心什么话题"，而是"有什么事情你翻来覆去想过很多遍"
> 追问 A："这个问题，你大概什么时候开始问自己的？"
> 追问 B："它有没有被部分解决过？是什么阻止了它被彻底解决？"
> 追问 C："这个问题如果没有答案，对你来说意味着什么？"
>
> **反模式探测：** 如果用户说"我没什么特别的问题"，追问："那最近有什么事让你觉得'哪里不对'？那个'哪里不对'最后指向了什么？"

> Q2-2：「你最近在思考什么？或者——什么事情最让你困惑？」
>
> 引导：困惑比答案更能暴露问题意识。
> 追问 A："这个困惑，最让你不舒服的一点是什么？"
> 追问 B："这个问题和 Q2-1 的那个，是同一个问题的不同层面，还是两个不同的问题？"
> 追问 C："你试过回答它吗？你的答案是什么？"
>
> **反模式探测：** 如果用户说"我想通了"，追问："那个答案，你是真正接受了，还是只是找到了一种说法让自己好受？"

**提取原则：**
- 区分"话题"和"问题"——前者是领域，后者是具体的、反复出现的疑问
- 写出这个问题对用户的意义，不只是问题本身
- 如果用户同时提出多个问题，识别哪个是最深层的（其他问题是否是这个问题的变体）
- **问题演进路径**：这个问题从第一次出现到现在，有没有变过？怎么变的？

**层后合成确认：**

```
Layer 2 确认单：

  ❓ 你一直在问的问题：
    {问题核心——一句话}

  为什么这个问题对你重要：
    {对你意味着什么}

  开始时间：{什么时候/什么触发}
  问题演进：{从第一次到现在，这个问题有没有变过？怎么变的？}
  部分解决过吗？{是/否}——是什么阻止了彻底解决？

  确认进入 Layer 3？（确认 / 补充 / 跳过）
```

#### Layer 3 — 你说话的方式

##### Layer 3 — 表达风格的场景差异表

| 场景 | 表达特征 |
|------|---------|
| **微信文字** | 是否爱发表情包 / 是否爱用语音 / 回复速度 / 是否会打很长的小作文 |
| **当面闲聊** | 话题主导还是跟随 / 会不会主动开启话题 / 说话被打断时的反应 |
| **正式场合（汇报/评审）** | 变得更书面化还是更口语化 / 是否会紧张 / 是否会提前准备发言稿 |
| **私下/熟人面前** | 最放松时的状态 / 会不会开玩笑 / 会聊什么类型的话题 |
| **被质疑时** | 沉默/反驳/解释/转移 / 第一反应是什么 |
| **想表达不满时** | 直接说/绕弯子/阴阳/冷暴力 / 会用什么方式让对方知道 |
| **写重要文档时** | 长文详细/短平快 / 是否会反复修改 / 最大的风格特点是什么 |
| **邮件** | 长邮件还是短邮件 / 会不会 CC 很多人 / 会不会用大段文字解释背景 |

##### Layer 3 — 核心问题

> **Q3-热身**（新增）：「先问一个简单的——你觉得你说话偏理性还是偏感性？就是那种，你写完一段话之后，是先检查逻辑还是先检查感受？」
>
> 引导：这个问题建立"表达是有迹可循的"的基础感。回答"理性"的用户，可能会在后续暴露"我的问题是说完就算了"；回答"感性"的用户，可能会在后续暴露"我说话经常情绪先行然后后悔"。
>
> **设计目的：** Layer 3 的旧版直接问口头禅，但有些用户不认为自己是"有风格"的人。这个热身让所有人都能找到一个锚点进入表达讨论。

> Q3-1：「你说话有没有什么习惯性的表达方式？比如某些词、某些句式，是你经常用的？」
>
> 引导：如果用户说"没有"，问"那别人有没有模仿过你说话？模仿的是什么？"
> 追问："那个表达，你每次说的时候是真的在想那个意思，还是已经变成了一个无意识的习惯？"
>
> **反模式探测：** 如果用户说"我没什么口头禅"，追问："那你写微信的时候呢？有没有哪个词是你用得特别多的？"

> Q3-2：「有没有一种情况，你必须说一件事，但你会故意绕个弯子说？」
>
> 引导：暴露表达策略，这比口头禅更能定义一个人的说话方式。
> 追问 A："你绕弯子的时候，听的人一般有没有察觉？"
> 追问 B："有没有你绕了弯子、但其实对方完全听懂了的时刻？"
> 追问 C："你有没有发现自己在什么情况下会绕弯子？那个弯子是在保护什么——保护自己，还是保护对方，还是保护这段关系？"
>
> **反模式探测：** 如果用户说"我说话很直接"，追问："直接的人也会有些话不想直说。你有吗？什么时候？"

> **Q3-3（新增追问）：「你有没有一个场合，说的时候和你想的不一样？就是你明明是那个意思，但说出来之后你自己都觉得'不对，我说漏了'？」
>
> 引导：这个问题揭示了"表达失误"的模式——有些人说漏嘴是因为太直接，有些人是因为想保护什么。追问："后来你怎么补救的？"
>
> **设计目的：** Layer 3 的旧版 Q3-3 只问"微信和当面是否一样"，新版更深入地追问"说漏"的时刻，帮助用户识别自己的表达风险点。

> Q3-4：「你发微信和当面说话，风格一样吗？」
>
> 引导：捕捉不同场景下的表达差异。
> 追问 A："有没有一种场合，你说话比平时更'端着'？是什么场合？"
> 追问 B："在什么人面前，你说话最放松？那种放松是什么状态？"
> 追问 C："有没有一个人，你是跟他当面不聊但微信会聊很多？为什么？"

#### Layer 4 — 你真正在乎什么

##### Layer 4 — 价值观行为翻译表

| 标签 | 翻译为行为规则 |
|------|-------------|
| **诚信** | "说过的话一定做到"——但这意味着什么？是会提前告知变化，还是宁可硬撑也不食言？ |
| **自由** | 不喜欢被管束——但你会为什么妥协？你愿意用自由换什么？ |
| **成长** | 重视个人发展——但这意味着你愿意放弃稳定吗？你会为什么放弃成长？ |
| **关系** | 重视人际关系——但你的关系优先级是什么？家人/朋友/同事/自己？ |
| **意义** | 重视工作的意义——但"有意义"对你来说意味着什么？被认可还是真的在改变什么？ |
| **公平** | 重视公正——但你愿意为公平付出什么代价？你会因为不公平而离开吗？ |
| **影响力** | 重视自己的影响力——但你通过什么方式施加影响？写东西/演讲/日常对话？ |
| **独立** | 不喜欢依赖别人——但这会让你回避合作吗？你会不会因为"想自己搞定"而拒绝帮助？ |
| **美** | 对美有强烈追求——但这会影响你做决定吗？你会为了美放弃效率吗？ |
| **真相** | 哪怕真相很难听也要说——但你会在什么时候选择不说？那个"不说"是因为什么？ |

##### Layer 4 — 核心问题

> **Q4-热身**（新增）：「在你心里，有没有一件事是你绝对不会为了任何人放弃的？哪怕所有人都觉得你傻，你也不会放弃的那种？」
>
> 引导：这个问题是 Layer 4 的核心锚点。比起"你最在乎什么"，用户更容易回答"你最不愿意放弃什么"。
> 追问 A："有没有人曾经试图让你放弃？你怎么回应的？"
> 追问 B："那个不能放弃的东西，是什么？为什么它对你这么重要？"
>
> **设计目的：** Layer 4 的旧版开场问"最近一年放弃了什么"，但这需要回忆。新版热身直接问"不能放弃什么"，让用户先确定核心，再通过"放弃了什么"来验证。

> Q4-1：「最近一年，你为了做 A 放弃了 B——放弃了什么？A 是什么？」（或反过来）
>
> 引导：你的时间流向就是你的价值观排序。
> 追问 A："放弃的那一刻，你心里有挣扎吗？还是很快就做了决定？"
> 追问 B："现在回头看，你觉得那个选择对吗？有没有哪个时刻怀疑过自己？"
>
> **反模式探测：** 如果用户说"我最近没什么放弃"，追问："那最近几年呢？换工作、搬家、结束一段关系——有没有哪个决定你事后觉得'我其实放弃了不少'？"

> Q4-2：「有没有一件事，你最近几年一直在做，尽管所有人都在说它没意义？」
>
> 引导：这暴露了内生的价值驱动，不是外部认可驱动的。
> 追问 A："没有人说它有意义，你为什么还在做？"
> 追问 B："有没有想过放弃？为什么没有？"
> 追问 C："这件事，你在跟谁做？如果那个人不在了，你还会继续吗？"
>
> **反模式探测：** 如果用户说"没有这种事"，追问："那你有没有一个事情，你做它的时候不是为了任何回报，纯粹是因为'不做会难受'？"

> **Q4-3（新增追问）：「你有没有一个时候，是被逼着在两个都很重要的东西里选一个？你选了哪个？后来呢？」
>
> 引导：这个问题帮我们找到用户真正的价值底线——在两难中选了哪个，意味着哪个对他来说其实是第二位的。
> 追问 A："那个你没选的东西，后来你有没有后悔？还是你觉得值得？"
> 追问 B："如果你现在重新选一次，你会改变吗？"

> Q4-4：「你最近一次跟一个人吵架（或发生冲突），是因为什么？」
>
> 引导：价值观冲突往往在愤怒中暴露。
> 追问 A："回过头看，你生气的真正原因是什么？是对方伤害了你，还是对方让你失望了？"
> 追问 B："这个冲突，后来解决了吗？怎么解决的？"
>
> **反模式探测：** 如果用户说"我不怎么吵架"，追问："那你有没有一个时刻，你特别想发火但忍住了？那个忍住，是出于什么——保护关系、等待时机、还是觉得发火没用？"

**提取维度：**
- 价值观排序列表（从行为中推断，不是自我描述）
- 每个价值的含义（你对它的理解，不是一般人对它的理解）
- 当前最让你痛苦的价值冲突是什么

#### Layer 5 — 你的不可替代时刻

##### Layer 5 — 不可替代性来源分类表

| 来源 | 描述 | 示例 |
|------|------|------|
| **看见别人看不见的** | 你因为经历/背景/视角特殊，能看到别人忽略的信息 | "别人看数据下降要拉新，我知道这个时候做留存才是对的——因为我见过太多次同样的模式" |
| **在乎别人不在乎的** | 你的情感优先级和大多数人不一致 | "所有人都在关心能不能按时上线，我在关心上线后用户会不会骂我们" |
| **做得到别人做不到的** | 不是技能，是某种行为模式 | "别人遇到冲突就回避，我反而会在这个时候跟对方单独约聊" |
| **独特判断框架** | 你做判断的方式和别人不同 | "所有人都在追热点，我觉得追热点的人都短视——我会去研究那些没人在讨论的东西" |

##### Layer 5 — 核心问题

> Q5-1：「有没有一个判断或决定，你知道如果是你来做，会跟别人不一样？是什么不一样？」
>
> 引导：描述那个判断本身。
> 追问 A："那个不同，是因为你看到了别人没看到的什么？还是因为你关心别人不关心的什么？"
> 追问 B："那个不同，最后有没有被证明是对的？"
>
> **反模式探测：** 如果用户说"好像没有"，追问："有没有一个时刻，你知道你的判断会跟大多数人不一样，但你坚持了自己的判断？后来怎么样了？"

> Q5-2：「你有没有一个习惯性的反应，是别人不会这样做的？」
>
> 引导：这暴露了表达和行为上的独特之处。
> 追问："你意识到自己和别人不一样的时候，是什么感觉？"
>
> **反模式探测：** 如果用户说"我觉得我跟大家都一样"，追问："那你观察一下你身边最熟的人——他们发邮件的方式、他们开会的习惯，有没有哪个你绝对不会用同样的方式做的？"

> Q5-3：「如果有一个人要假装是你，他最可能在哪个地方露馅？」（**重构**)
>
> 旧版问法："如果有一个人完全复制了你的知识和能力，但他缺了什么才不是你？"
> 新版更具体：因为"复制知识和能力"是假设情境，"露馅"来自真实被观察经验，更容易回答。
>
> 追问 A："是你说的话？还是你做决定的方式？还是你不会做的事情？"
> 追问 B："有没有人当面说过'你跟我想的不一样'？或者背后议论你的时候，说你'总是会……'？"
> 追问 C："这个东西，能被教吗？还是只能被经历？"
>
> **反模式探测：** 如果用户说"应该是经验吧"，追问："经验可以被复制。你见过有人复制了你的经验但结果完全不对吗？那个不对里面，缺的是什么？"

**提取：** 3-5 个"不可替代时刻"。

#### Layer W — 工作流与生活节奏

##### W.0 工作风格行为翻译表

| 职业类型 | 典型工作流特征 |

| **研发工程师** | 启动：先想架构还是先写代码；调试：喜欢加日志还是设断点；代码审查：关注什么类型的问题 |
| **设计师** | 启动：先找参考还是先发呆；改稿：改几版才满意；交付：给源文件还是只给成品 |
| **运营** | 启动：先看数据还是先拍脑袋；执行：喜欢 SOP 还是随机应变；复盘：用什么形式记录 |
| **销售/BD** | 启动：先建立关系还是先谈正事；推进：用什么方式跟进；成交：关键转折点是什么 |
| **管理层** | 启动：自己决策还是收集信息；执行：授权还是亲自盯；评估：用过程指标还是结果指标 |
| **独立顾问/自由职业** | 启动：怎么获客；定价：按时间还是按项目；交付：一个人做还是外包一部分 |

##### Layer W — 核心问题

>

>
> **设计目的：** Layer W 的问题容易让人觉得在"介绍工作经验"。这个热身把话题锚定在"感受"上，让用户从"我平时怎么做"切换到"我其实有自己的节奏和偏好"，为后面的具体问题做好铺垫。

## 进化模式 A：追加文件（**增强版：新增语义冲突检测**）

用户提供新的对话、文件或口述信息时：

**Step A1：读取新内容**
- 文件：用 `Read` 读取
- 口述：直接作为文本原材料

**Step A2：读取现有文件**
- 用 `Read` 读取 `~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/identity.md`
- 用 `Read` 读取 `~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/persona.md`

**Step A3：分析增量内容并归入 Layer**

| 新内容类型 | 归入 |
|-----------|------|
| 新的绝对红线 / 绝不做的事 | → Layer 0 |
| 新的决策案例 / 权衡方式 | → Layer 1 |
| 新发现的问题意识 / 困惑 | → Layer 2 |
| 新的表达习惯 / 口头禅 | → Layer 3 |
| 新的取舍行为 / 价值冲突 | → Layer 4 |
| 新的不可替代时刻 | → Layer 5 |
| 新的工作方式 / 方法论 / 输出偏好 / 踩过的坑 | → Layer W |
| 新的核心矛盾或真实弱点描述 | → identity.md 核心矛盾节 / 真实弱点节 + persona.md 对应节 |

**Step A4：语义冲突检测（核心步骤）**

对每个待追加的 Layer 内容，执行以下检测：

**检测一：重复检测**
- 计算新旧内容的语义相似度（高 overlap → 直接告知"已存在"）
- 低于 80% 语义重叠的新内容才进入追加流程

**检测二：覆盖检测**
- 新内容与现有内容指向同一主题但更准确/更完整 → 标记为"覆盖候选"
- 向用户展示覆盖前后对比

**检测三：冲突检测**
- 新内容与现有内容在事实层面矛盾 → 标记为"冲突"
- 触发冲突确认流程

**Step A5：Merge 策略（五选一）**

| 规则 | 条件 | 操作 |
|------|------|------|
| **规则 1 - 追加新子项** | 新内容是现有 Layer 下的新子项 | 直接追加，不改动已有内容 |
| **规则 2 - 替换/覆盖** | 新内容与现有内容指向同一件事但更准确/更完整 | 展示覆盖前后对比，用户确认后替换 |
| **规则 3 - 新增 Layer** | 新内容落在原本为空的 Layer | 新建该 Layer 节，标记"新增于 vN+1" |
| **规则 4 - 重复内容** | 低于 20% 语义新增（与现有内容高度重合） | 忽略，告知用户"已存在，无需追加" |
| **规则 5 - 冲突** | 新内容与现有内容矛盾 | 暂停，输出冲突提示，等用户确认 |

**Step A6：冲突确认模板（**按实际场景分三种情况**）：**

> **情况一：既有冲突也有覆盖候选**
```
⚠️ 发现需要确认的情况：

--- 冲突 {N} ---
  Layer {X} - {Layer 名称}
  ❌ 现有(v{vN})：{现有描述}
  ✅ 新发现：{新内容描述}

  以哪条为准？
    [1] 保留现有
    [2] 更新为新内容
    [3] 两者都保留，标注适用场景不同

--- 覆盖候选 {N} ---
  Layer {Y} - {Layer 名称}
  现有：{现有描述}
  新内容更完整：{新内容描述}
  直接覆盖？（确认 / 保留现有）
```

> **情况二：只有冲突，没有覆盖候选**
```
⚠️ 发现 {N} 处冲突：

--- 冲突 {N} ---
  Layer {X} - {Layer 名称}
  ❌ 现有(v{vN})：{现有描述}
  ✅ 新发现：{新内容描述}

  以哪条为准？
    [1] 保留现有
    [2] 更新为新内容
    [3] 两者都保留，标注适用场景不同
```

> **情况三：只有覆盖候选，没有冲突**
```
📋 发现 {N} 处可优化的内容：

--- 覆盖候选 ---
  Layer {Y} - {Layer 名称}
  现有：{现有描述}
  新内容更完整：{新内容描述}
  直接覆盖？（确认 / 保留现有）
```

**Step A7：备份当前版本**
```bash
cp ~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/identity.md \
   ~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/versions/identity_v{vN}.md
cp ~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/persona.md \
   ~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/versions/persona_v{vN}.md
```

**Step A8：用 `Edit` 写入增量内容**

对 `identity.md`：追加/替换对应 Layer 节；在 `## 进化记录` 中追加一行；在 `## Correction 记录` 中注明来源。

对 `persona.md`：追加/替换对应 Layer 节；如有新增反模式，追加到反模式节；核心矛盾和真实弱点节同步更新。

**Step A9：重新生成完整 SKILL.md**

**Step A10：更新用户 profile meta.json**
- 更新 `~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/meta.json`
- version 从 `v{N}` 升级为 `v{N+1}`
- updated_at 更新为当前时间

**Step A11：展示追加影响摘要**

向用户展示：

```
     Layer {N} 追加了 {描述}
```

## 进化模式 B：对话纠正（**增强版：新增纠正链连锁检查**）

- "这不对" / "不对" / "错了"
- "我不会这样" / "我不是这样的"
- "我应该是" / "我其实是"
- "这不像我"

**Step B2：追问澄清（一次）**
如果用户说得模糊，追问：

```
我理解你在纠正的是：{你的理解}
```

| 纠正内容 | 归入文件 |

| 决策方式 / 权衡逻辑 | → identity.md Layer 1 |

| 说话方式 / 口头禅 / 表达习惯 | → identity.md Layer 3 + persona.md |

| 反模式（"这不像{name}"） | → persona.md 反模式节 |

**Step B4：展示纠正前后**

```
  Layer {N} - {Layer 名称}
```

```bash
cp ~/.openclaw/workspace/skills/irreplaceable-you/"{slug}"/persona.md \
```

**Step B7：纠正链连锁检查（核心步骤）**

此纠正会影响其他层吗？按以下连锁规则检查：

| 如果纠正了 | 则自动检查 |
|-----------|-----------|
| Layer 0（红线） | Layer 1（决策是否与红线冲突）、Layer 4（价值观是否被红线约束） |
| Layer 1（决策） | Layer 0（是否违反红线）、Layer 5（决策风格是否影响不可替代时刻） |
| Layer 2（问题意识） | Layer 1（问题是否影响决策）、Layer 5（独特问题是否形成独特判断） |
| Layer 3（表达） | persona.md 表达节（是否需要同步更新） |
| Layer 4（价值观） | Layer 1（价值观是否影响决策权衡）、Layer 0（红线是否体现价值观） |
| Layer 5（不可替代） | Layer 1（判断框架是否影响决策） |
| Layer W（工作流） | persona.md 工作方式节（是否需要同步更新） |
| 核心矛盾 | Layer 4（矛盾是否与价值排序一致）、Layer 5（矛盾是否影响不可替代时刻） |
| 真实弱点 | Layer 1（弱点是否影响决策方式）、Layer 3（弱点是否影响表达） |

连锁检查后向用户展示：

```

```

**Step B8：correction 数量上限**

correction 记录每文件最多保留 50 条。超出时，将语义相近的 correction 合并归纳为 1 条，优先保留最新表述。告知用户："已将 {N} 条相似规则合并为 {M} 条"。
