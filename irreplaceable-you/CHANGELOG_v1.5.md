### 核心改进

#### 1. Step 1 Q3 重构：「钩子 + 递进序列」设计

**旧版问题**：直接问"用一句话描述一个你反复出现的行为模式"，用户容易给出宽泛标签。

**新版设计**：
- **热身钩子**：「先不说为什么——最近有没有一件事，你做了但事后感觉有点奇怪？」
- **展开追问**：背景 → 代价 → 后果
- **提炼追问**：这说明你有什么不一样的地方？
- **行为探测矩阵**：如果用户说得太抽象，自动追问具体场景

参考：yourself-skill2 的"具体引导问题序列"（8 个引导问题）+ ex-skill2/colleague-skill 的"行为画像"设计。

#### 2. Step 1.5 重构：「行为探测式」Layer 意向选择

**旧版问题**：列出 7 个 Layer 让用户打分/选"深挖/快过/跳过"，容易变成机械操作。

**新版设计**（参考 yourself-skill2）：
- 不打分，用三句话描述自己："哪个最清楚 / 哪个从没想过但重要 / 哪个最不想跳过"
- Layer 编号直接回答，无需量表操作
- 行为描述取代量表操作，减少填表感

#### 3. 进化追加模式增强：语义冲突检测

新增 Step A4 **语义冲突检测**，填补旧版"直接追加"的漏洞：
- **检测一：重复检测**（<20% 语义新增 → 忽略）
- **检测二：覆盖检测**（更准确/更完整 → 覆盖候选）
- **检测三：冲突检测**（事实层面矛盾 → 冲突）

参考：colleague-skill / ex-skill2 / yourself-skill2 的 Merger 逻辑 + boss-skills2 的结构化覆盖/追加设计。

#### 4. 进化纠正模式增强：纠正链连锁检查

新增 Step B7 **纠正链连锁检查**：
- 纠正某个 Layer 时，自动检查哪些其他 Layer 需要联动调整
- 按「Layer 依赖关系矩阵」执行自动推理
- 用户一次确认全部联动变更，而非逐层纠正

参考：colleague-skill / yourself-skill2 的 Correction Handler 逻辑。

#### 5. 整体设计改进：减少填表感

- 开场白精简：去掉大段说明，直接给对比示例
- Layer 合成确认表统一简化：每层一个确认块，不重复展示
- Q3 答好标准：结尾给对比示例，而非在开头展示
- 对话质感：增加"引导"、"追问"类型的区分，让 AI 知道何时追问何时确认

## 详细变更

### 1. Layer 0 标签翻译表（全新章节）

**来源灵感：** colleague-skill（titanwings，6000+ GitHub stars）的 Part B Layer 0 部分。

**设计背景：** 用户常说"INTJ"、"狮子座"、"数据产品经理"这类标签，但这些标签本身对 AI 没有可操作意义。colleague-skill 的 Layer 0 标签翻译方案（把"甩锅高手"翻译为"遇到问题第一反应是找外部原因"）提供了极好的示范。

**新增内容：**
- **MBTI 类型翻译**（7 种类型）：INTJ / ENFP / INFJ / ENTJ / INFP / ISTJ / ESTP
- **星座翻译**（7 种）：狮子座 / 白羊座 / 天蝎座 / 双子座 / 处女座 / 射手座 / 金牛座
- **职业标签翻译**（7 种）：数据产品经理 / 运营 / 研发工程师 / 设计师 / 销售BD / 创始人CEO / 咨询顾问
- **使用说明**：翻译为 Layer 0 的初始假设，需用户确认是否与实际相符

**示例（INTJ狮子座，数据产品经理）：**
```
INTJ：遇到问题第一反应是画框架；评价方案先问"逻辑自洽吗"；不喜欢没有预案的讨论；……
狮子座：需要被认可，尤其是在公开场合；喜欢做主角而不是配角；被忽视比被批评更难受；……
数据产品经理：判断一个需求，先问"有没有数据"；不信任没有数字支撑的结论；……
```

### 2. Layer W：工作流与 Capability（全新章节）

**来源灵感：** colleague-skill 的 Part A Work Skill 部分（responsibilities, technical specs, code standards, CR preferences, experience knowledge base, work processes）。

**设计原则：** 原始 irreplaceable-you 的 6 层结构回答了"who you are"（你是谁），但没有回答"how you work"（你是怎么工作的）。Layer W 将 skill 从"身份蒸馏"扩展为"身份 + 能力方式蒸馏"。

**子维度：**

| 维度 | 说明 | 对应问题 |
|------|------|---------|
| **工作流** | 启动方式、执行节奏、收尾方式 | QW-1, QW-2 |
| **输出偏好** | 文档风格、演示风格、沟通偏好 | QW-3, QW-4 |
| **核心方法论** | 反复使用的思维/工作模式 | QW-5 |
| **踩过的坑** | 教训与改变 | QW-6 |
| **精力与节奏** | 高效时段、充电方式、工作边界 | QW-7（可选） |

**7 个新增问题：**
- QW-1：接到任务怎么开始？
- QW-2：做完怎么交付？
- QW-3：写文档什么风格？
- QW-4：开会扮演什么角色？
- QW-5：有什么固定方法论？
- QW-6：踩过最大的坑是什么？
- QW-7（可选）：生活节奏是什么样的？

## 灵感来源：colleague-skill 结构分析

### colleague-skill 是什么

colleague-skill 是一个由 titanwings 开发的开源 AI Agent Skill（GitHub 6000+ stars），旨在让 AI 能够像某个同事一样工作。

### colleague-skill 的核心结构

**Part A: Work Skill（工作技能层）**
- Responsibilities（职责范围）
- Technical specs & code standards（技术规范和代码标准）
- CR preferences（代码审查偏好）
- Experience knowledge base（经验知识库）
- Work processes（工作流程）

**Part B: Persona（人格层，5层）**
- Layer 0：Hard Rules（硬规则）
- Layer 1：Identity（身份认同）
- Layer 2：Expression（表达方式）
- Layer 3：Decision（决策方式）
- Layer 4：Interpersonal（人际风格）
- Layer 5：Boundaries（边界）

### irreplaceable-you v1.5 的吸收方式

| colleague-skill 概念 | 在 irreplaceable-you v1.5 中的对应 |
|---------------------|-------------------------------------|
| Work Skill（Part A） | → Layer W（工作流与 Capability） |
| Layer 0 Hard Rules | → Layer 0 绝对红线（原有）+ 标签翻译表（新增） |
| Layer 1–5 Identity/Boundaries | → Layer 1–5（对应映射，略有调整） |
